Introducción a HTML

¿Qué es HTML?

HTML significa HyperText Markup Language. Es el lenguaje de marcado que se utiliza para definir la estructura de una página web.

Cuando una persona abre un sitio en el navegador, gran parte de lo que ve comienza como un documento HTML. Ese documento le indica al navegador qué partes del contenido son títulos, párrafos, enlaces, imágenes, listas, tablas, formularios o secciones.

HTML no es un lenguaje de programación. No se usa para expresar algoritmos, decisiones o repeticiones como ocurre en lenguajes como JavaScript, Python o Java. Su función principal es describir y organizar información.

En desarrollo web suele trabajarse con tres tecnologías principales:

  • HTML: estructura y significado del contenido
  • CSS: presentación visual
  • JavaScript: comportamiento e interactividad

Una forma simple de pensarlo es esta:

  • HTML dice qué es cada cosa
  • CSS dice cómo se ve
  • JavaScript dice cómo se comporta

¿Por qué se dice que es un lenguaje de marcado?

Se lo llama lenguaje de marcado porque usa marcas o etiquetas para delimitar partes del contenido y darles un significado.

Por ejemplo, si escribimos:

<p>Esto es un párrafo.</p>

el texto sigue siendo el mismo, pero ahora el navegador sabe que debe tratarlo como un párrafo.

Del mismo modo:

<h1>Programación III</h1>

indica que ese texto es un encabezado principal.

Primer ejemplo

Un documento HTML mínimo puede verse así:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Mi primera página</title>
  </head>
  <body>
    <h1>Hola mundo</h1>
    <p>Este es mi primer documento HTML.</p>
  </body>
</html>

Aunque todavía no entendamos todas las etiquetas, este ejemplo ya muestra una idea importante: HTML organiza el contenido dentro de una estructura jerárquica.

Estructura general de un documento HTML

<!DOCTYPE html>

La declaración <!DOCTYPE html> indica al navegador que el documento está escrito siguiendo el estándar actual de HTML.

  • Debe aparecer al principio del archivo
  • No es una etiqueta HTML común
  • Ayuda al navegador a interpretar correctamente el documento

<html>

La etiqueta <html> envuelve a todo el documento HTML.

Suele incluir el atributo lang para indicar el idioma principal del contenido:

<html lang="es"></html>

Esto mejora la accesibilidad, el trabajo de los lectores de pantalla y ciertas funciones del navegador.

<head>

La sección <head> contiene metadatos, es decir, información sobre el documento que no forma parte del contenido visible principal.

Por ejemplo:

  • el título de la página
  • la codificación de caracteres
  • metadatos para buscadores
  • enlaces a hojas de estilo
  • referencias a scripts

Un ejemplo típico:

<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Mi primera página</title>
</head>

<body>

La sección <body> contiene el contenido que la persona usuaria ve en la pantalla:

  • títulos
  • párrafos
  • imágenes
  • enlaces
  • listas
  • tablas
  • formularios

Elementos, etiquetas y atributos

Estos tres conceptos se relacionan, pero no son exactamente lo mismo.

Etiqueta

Una etiqueta es la marca escrita entre corchetes angulares.

Por ejemplo:

<p></p>

Elemento

Un elemento HTML es la unidad completa: etiqueta de apertura, contenido y etiqueta de cierre.

<p>Esto es un párrafo.</p>

En este caso, el elemento es todo el bloque.

Atributo

Un atributo agrega información adicional al elemento.

<a href="https://www.google.com">Ir a Google</a>

Aquí:

  • a es el nombre del elemento
  • href es un atributo
  • "https://www.google.com" es el valor de ese atributo

Los atributos suelen escribirse:

  • dentro de la etiqueta de apertura
  • con el formato nombre="valor"
  • entre comillas

Anatomía de un elemento HTML

En muchos elementos podemos distinguir:

  • etiqueta de apertura
  • contenido
  • etiqueta de cierre
  • atributos

Por ejemplo:

<p class="destacado">Este texto pertenece a un párrafo.</p>

Elementos contenedores y elementos vacíos

Algunos elementos contienen otros elementos o texto. Por ejemplo:

<p>Este párrafo contiene texto.</p>

Otros elementos no envuelven contenido y se usan de manera independiente. Se los suele llamar elementos vacíos.

Ejemplos frecuentes:

  • <br />
  • <img />
  • <meta />
  • <input />

Por ejemplo:

<img src="foto.jpg" alt="Una foto" />

Anidación

Los elementos HTML pueden colocarse unos dentro de otros. A eso se lo llama anidación.

<body>
  <p>Este párrafo está dentro del body.</p>
</body>

También pueden anidarse varios niveles:

<section>
  <article>
    <h2>Noticia</h2>
    <p>Contenido de la noticia.</p>
  </article>
</section>

Regla importante

El elemento que se abre primero debe cerrarse último.

Correcto:

<p>Texto con <strong>énfasis importante</strong>.</p>

Incorrecto:

<p>Texto con <strong>énfasis importante</p></strong>

Cuando las etiquetas se cierran en un orden incorrecto, el documento queda mal formado y el navegador puede interpretarlo de manera inesperada.

Espacios, saltos de línea e indentación

En HTML, varios espacios seguidos y muchos saltos de línea suelen colapsarse al mostrarse en pantalla. Esto significa que el navegador normalmente no respeta el formato visual del código fuente como si fuera texto literal.

Por ejemplo:

<p>Una oración Otra oración Tercera oración</p>

se visualizará como una sola línea de texto continuo.

La indentación y los saltos en el archivo sirven sobre todo para que el código sea más legible para quien lo escribe.

Elementos de uso general

Encabezados

Los encabezados se definen con las etiquetas <h1> a <h6>.

  • <h1> representa el encabezado más importante
  • <h6> representa el de menor jerarquía
<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección interna</h3>

No conviene usarlos solo porque "se ven más grandes" o "más chicos". Deben reflejar la jerarquía lógica del contenido.

Párrafos

Los párrafos se definen con la etiqueta <p>.

<p>Esto es un párrafo.</p>
<p>Esto es otro párrafo.</p>

Cada <p> representa una unidad de texto separada.

Saltos de línea

La etiqueta <br /> inserta un salto de línea sin crear un nuevo párrafo.

<p>
  Línea 1<br />
  Línea 2<br />
  Línea 3
</p>

No debe usarse para separar bloques enteros de contenido ni para "acomodar" el diseño. Para eso se utilizan otras etiquetas y, más adelante, CSS.

Texto preformateado

La etiqueta <pre> permite mostrar texto respetando espacios y saltos de línea.

<pre>
Línea 1
  Línea 2 con indentación
    Línea 3 con más indentación
</pre>

Es útil cuando se quiere mostrar:

  • código
  • salidas de consola
  • texto donde el espaciado tiene significado

Formato básico de texto

HTML ofrece etiquetas para marcar distintos tipos de énfasis o significado en el texto.

<p>
  Texto <strong>importante</strong>, texto <em>enfatizado</em> y texto
  <mark>resaltado</mark>.
</p>

Algunas etiquetas frecuentes son:

  • <strong>: importancia fuerte
  • <em>: énfasis
  • <mark>: texto resaltado
  • <small>: texto secundario o de menor importancia
  • <sup>: superíndice
  • <sub>: subíndice
  • <del>: contenido eliminado
  • <ins>: contenido insertado

También existen <b> e <i>, pero en general conviene priorizar etiquetas con más significado semántico, como <strong> y <em>.

Enlaces

Los enlaces se definen con la etiqueta <a>.

<a href="https://www.google.com">Ir a Google</a>

El atributo href indica el destino.

Enlace absoluto

Apunta a una dirección completa:

<a href="https://www.uncoma.edu.ar">Sitio de la universidad</a>

Enlace relativo

Apunta a otro recurso dentro del mismo sitio o proyecto:

<a href="contacto.html">Ir a contacto</a>

Abrir en otra pestaña

<a href="https://www.github.com" target="_blank">GitHub</a>

target="_blank" suele abrir el recurso en otra pestaña o ventana.

Buenas prácticas para enlaces

  • El texto del enlace debe describir a dónde lleva
  • Conviene evitar frases ambiguas como "hacé clic acá"
  • Si el enlace abre un recurso externo, puede ser útil aclararlo en el texto

Imágenes

Las imágenes se insertan con la etiqueta <img />.

<img src="logo.jpg" alt="Logo de la materia" width="120" />

Algunos atributos frecuentes son:

  • src: ubicación de la imagen
  • alt: texto alternativo
  • width: ancho
  • height: alto

¿Para qué sirve alt?

El atributo alt es importante por varios motivos:

  • mejora la accesibilidad
  • describe la imagen si no puede cargarse
  • ayuda a tecnologías de asistencia

El texto alternativo debe describir la imagen cuando esa descripción aporta información relevante.

Rutas absolutas y relativas

Cuando un atributo apunta a un recurso, como ocurre con href o src, la ruta puede ser absoluta o relativa.

Ruta absoluta

Incluye la dirección completa:

<img src="https://www.ejemplo.com/imagenes/logo.png" alt="Logo" />

Ruta relativa

Se interpreta a partir de la ubicación del archivo actual:

<img src="imagenes/logo.png" alt="Logo" />

En proyectos reales, las rutas relativas son muy comunes para enlazar archivos del mismo sitio.

Listas

Lista desordenada

Se crea con <ul> y cada elemento se define con <li>.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Lista ordenada

Se crea con <ol>.

<ol>
  <li>Abrir el editor</li>
  <li>Crear el archivo HTML</li>
  <li>Escribir la estructura base</li>
</ol>

Las listas ordenadas son útiles cuando el orden de los ítems importa.

Contenedores genéricos: div y span

Existen elementos que no describen por sí mismos un tipo específico de contenido, pero sirven para agrupar partes del documento.

<div>

Es un contenedor genérico de bloque.

<div>
  <h2>Noticias</h2>
  <p>Contenido de la sección.</p>
</div>

<span>

Es un contenedor genérico en línea.

<p>El resultado fue <span>aprobado</span>.</p>

En HTML moderno conviene usar primero elementos semánticos cuando existan, y dejar div y span para los casos en que no haya una etiqueta más expresiva.

HTML semántico

Decimos que un HTML es semántico cuando utiliza etiquetas que describen el rol del contenido.

En lugar de agrupar todo con div, HTML ofrece etiquetas como:

  • <header>
  • <nav>
  • <main>
  • <section>
  • <article>
  • <aside>
  • <footer>

Por ejemplo:

<body>
  <header>
    <h1>Portal de noticias</h1>
  </header>
 
  <main>
    <article>
      <h2>Título de la noticia</h2>
      <p>Contenido de la noticia.</p>
    </article>
  </main>
 
  <footer>
    <p>Copyright 2026</p>
  </footer>
</body>

Usar etiquetas semánticas mejora:

  • la legibilidad del código
  • la accesibilidad
  • la interpretación del documento por parte de buscadores y herramientas

Tablas

Las tablas sirven para representar datos tabulares, es decir, información organizada en filas y columnas.

<table>
  <thead>
    <tr>
      <th>Nombre</th>
      <th>Nota</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ana</td>
      <td>9</td>
    </tr>
    <tr>
      <td>Pedro</td>
      <td>7</td>
    </tr>
  </tbody>
</table>

Etiquetas principales:

  • <table>: tabla completa
  • <thead>: encabezado de la tabla
  • <tbody>: cuerpo de la tabla
  • <tr>: fila
  • <th>: celda de encabezado
  • <td>: celda de datos

No deben usarse tablas para maquetar páginas. Su uso correcto es representar datos.

Formularios

Los formularios permiten ingresar y enviar datos.

Un ejemplo básico:

<form>
  <label for="nombre">Nombre</label>
  <input id="nombre" type="text" name="nombre" />
 
  <label for="email">Email</label>
  <input id="email" type="email" name="email" />
 
  <button type="submit">Enviar</button>
</form>

Elementos frecuentes en formularios:

  • <form>: formulario completo
  • <label>: texto asociado a un campo
  • <input>: campo de entrada
  • <textarea>: área de texto multilínea
  • <select>: lista desplegable
  • <button>: botón

Aunque más adelante se verá en mayor profundidad, es importante entender que los formularios son una de las principales formas de interacción entre la persona usuaria y una aplicación web.

Comentarios

Los comentarios permiten dejar anotaciones en el código que no se muestran en la página.

<!-- Inicio del encabezado -->
<header>
  <h1>Mi sitio</h1>
</header>
<!-- Fin del encabezado -->

Pueden servir para:

  • organizar visualmente el archivo
  • explicar decisiones puntuales
  • marcar secciones durante el desarrollo

No conviene abusar de ellos si el código ya es suficientemente claro.

Buenas prácticas iniciales

Al empezar a escribir HTML conviene adoptar algunas costumbres desde el primer momento:

  1. Usar una estructura base completa con <!DOCTYPE html>, <html>, <head> y <body>
  2. Escribir las etiquetas en minúscula
  3. Indentar el código de forma consistente
  4. Cerrar correctamente los elementos
  5. Usar nombres de archivo simples y descriptivos
  6. Preferir etiquetas semánticas cuando existan
  7. Escribir texto alternativo útil en las imágenes
  8. No usar HTML para resolver problemas de diseño que corresponden a CSS

Resumen

En este primer contacto con HTML conviene quedarse con estas ideas:

  • HTML define la estructura de una página web
  • trabaja con elementos, etiquetas y atributos
  • los documentos tienen una estructura general común
  • los elementos pueden anidarse
  • existen etiquetas para títulos, párrafos, enlaces, imágenes, listas, tablas y formularios
  • HTML semántico ayuda a escribir documentos más claros y accesibles

Aprender HTML no consiste solo en memorizar etiquetas, sino en comprender cómo representar información de manera ordenada y con significado. Esa base será clave para todo lo que siga en desarrollo web.
