El modelo de caja

Introducción

Uno de los conceptos más importantes de CSS es el modelo de caja o box model.

Cuando el navegador representa muchos elementos HTML, los interpreta como si fueran cajas rectangulares. Esa idea no siempre es visible a simple vista, pero resulta fundamental para entender:

  • ¿por qué un elemento ocupa cierto espacio?
  • ¿por qué dos elementos quedan separados?
  • ¿cómo se calculan anchos y altos?
  • ¿cómo se construye la disposición visual de una página?

Comprender el modelo de caja evita muchos errores comunes al trabajar con espaciado, bordes, tamaños y alineación.

Las cuatro áreas de una caja

De manera simplificada, cada caja puede pensarse como formada por cuatro zonas:

  1. contenido
  2. padding
  3. borde
  4. margen

Por ejemplo:

.tarjeta {
  width: 300px;
  padding: 16px;
  border: 1px solid #cccccc;
  margin: 20px;
}

En ese caso, el elemento tiene un área de contenido, un espacio interno, un borde visible y una separación externa con respecto a otros elementos.

Área de contenido

El contenido es la parte central de la caja.

Allí se muestran:

  • el texto
  • las imágenes
  • otros elementos HTML anidados

Cuando se define width o height, en el modelo estándar de CSS esos valores se aplican, por defecto, al área de contenido.

Ejemplo:

.caja {
  width: 300px;
  height: 120px;
}

Esto no significa necesariamente que la caja completa mida 300 píxeles de ancho, porque luego pueden sumarse padding y border.

Padding

El padding es el espacio interno entre el contenido y el borde.

Sirve para que el contenido no quede "pegado" al borde del elemento.

.boton {
  padding: 10px 16px;
}

Si un botón tiene padding, el texto interior gana aire y la caja se ve más clara y cómoda para leer.

El padding:

  • aumenta el espacio interior
  • se encuentra dentro del borde
  • forma parte del tamaño visual del elemento

Border

El borde es la línea que rodea al contenido y al padding.

Se puede definir su:

  • grosor
  • estilo
  • color
.caja {
  border: 2px solid black;
}

El borde también ocupa espacio y, por lo tanto, influye en el tamaño final que se ve en pantalla.

Margin

El margin es el espacio externo que separa una caja de otras cajas.

.bloque {
  margin: 24px;
}

A diferencia del padding:

  • el padding está dentro de la caja
  • el margin está fuera de la caja

Esta diferencia es clave. Si se quiere separar el contenido del borde, se usa padding. Si se quiere separar un elemento de otros elementos, se usa margin.

Resumen visual del modelo

Puede pensarse así, desde adentro hacia afuera:

  1. contenido
  2. padding
  3. borde
  4. margin

O, expresado como idea:

el contenido vive dentro de la caja, el padding le da aire, el borde la delimita y el margin la separa de las demás

Ancho y alto en el modelo estándar

Una de las confusiones más frecuentes aparece con width y height.

En el modelo estándar de CSS:

  • width define el ancho del contenido
  • height define el alto del contenido

Si además se agregan padding y border, el tamaño total visible será mayor.

Por ejemplo:

.panel {
  width: 300px;
  padding: 20px;
  border: 2px solid gray;
}

En este caso, el ancho total visible será:

  • 300px de contenido
  • 20px de padding izquierdo
  • 20px de padding derecho
  • 2px de borde izquierdo
  • 2px de borde derecho

Total: 344px

Este detalle explica por qué a veces una caja "no entra" en el espacio esperado, aunque el ancho definido parezca correcto.

La propiedad box-sizing

Para simplificar el cálculo del tamaño real de una caja, CSS ofrece la propiedad box-sizing.

content-box

Es el valor predeterminado.

.caja {
  box-sizing: content-box;
}

Con este valor:

  • width y height afectan solo al contenido
  • padding y border se suman por fuera

border-box

Es muy usado en proyectos reales porque vuelve más predecible el cálculo del tamaño.

.caja {
  box-sizing: border-box;
  width: 300px;
  padding: 20px;
  border: 2px solid gray;
}

Con border-box, el ancho total de la caja sigue siendo de 300px, y dentro de ese valor quedan incluidos el contenido, el padding y el borde.

Por eso, en muchos proyectos se utiliza una regla general como esta:

* {
  box-sizing: border-box;
}

No cambia el significado del modelo de caja, pero sí cambia la forma en que se calcula el tamaño declarado.

Propiedades más usadas del modelo de caja

width y height

Definen el ancho y el alto del contenido, salvo que se esté usando box-sizing: border-box.

img {
  width: 200px;
  height: 150px;
}

padding

Define espacio interno.

button {
  padding: 10px 16px;
}

border

Permite establecer grosor, estilo y color del borde.

.caja {
  border: 2px solid black;
}

margin

Define espacio externo.

.bloque {
  margin: 24px;
}

Propiedades abreviadas

CSS permite escribir estas propiedades de forma abreviada.

Por ejemplo:

.caja {
  margin: 10px 20px 30px 40px;
}

Ese orden significa:

  1. arriba
  2. derecha
  3. abajo
  4. izquierda

También se pueden usar formas más cortas:

.caja {
  padding: 16px;
  margin: 10px 20px;
}

En ese caso:

  • padding: 16px aplica el mismo valor a los cuatro lados
  • margin: 10px 20px aplica 10px arriba y abajo, y 20px a izquierda y derecha

Ejemplo completo

<div class="tarjeta">Contenido de ejemplo</div>

.tarjeta {
  width: 320px;
  padding: 20px;
  border: 2px solid #58a6ff;
  margin: 24px;
  background-color: #f4f4f4;
}

En este ejemplo:

  • width define el ancho base del contenido
  • padding separa el texto del borde
  • border dibuja el contorno de la caja
  • margin separa la tarjeta del resto de los elementos

Relación con los elementos de bloque y en línea

El modelo de caja se entiende mejor cuando se lo relaciona con el tipo de elemento.

Los elementos de bloque, como <div> o <p>, suelen aceptar de manera natural:

  • ancho
  • alto
  • márgenes verticales
  • padding
  • bordes

Los elementos en línea, como <span> o <a>, también participan del modelo de caja, pero no siempre se comportan de la misma manera respecto del ancho, el alto o la distribución vertical.

Por eso, al estudiar el modelo de caja conviene conectarlo con la propiedad display.

Colapso de márgenes

Otro detalle importante es que los márgenes verticales de algunos elementos de bloque pueden colapsar.

Por ejemplo, si un elemento tiene margin-bottom: 20px y el siguiente tiene margin-top: 30px, no siempre se suman para dar 50px. En muchos casos, el navegador usa solo el valor mayor.

Este comportamiento puede sorprender al principio, porque no ocurre con el padding ni con los bordes.

No es necesario dominar todos los casos desde el comienzo, pero sí saber que el colapso de márgenes existe.

Errores frecuentes

  1. Pensar que width representa siempre el ancho total visible del elemento. En el modelo estándar, no es así.
  2. Usar margin cuando en realidad se necesita padding.
  3. Olvidar que el borde también ocupa espacio.
  4. Suponer que todos los elementos responden igual a width y height.
  5. No considerar box-sizing, lo que suele generar cálculos inesperados.

Síntesis

El modelo de caja describe cómo CSS representa muchos elementos HTML en forma de cajas rectangulares.

Cada caja está compuesta por:

  1. contenido
  2. padding
  3. borde
  4. margen

Entender este modelo permite:

  • calcular tamaños con mayor precisión
  • controlar mejor la separación entre elementos
  • diseñar interfaces más prolijas y predecibles
  • evitar errores comunes de espaciado

Es un tema central para todo lo que sigue en CSS: distribución, alineación, posicionamiento y diseño de interfaces.
