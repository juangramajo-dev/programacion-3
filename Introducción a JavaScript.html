Introducción a JavaScript

¿Qué es JavaScript?

JavaScript es un lenguaje de programación que se usa principalmente para agregar comportamiento a las aplicaciones web.

En una página o aplicación web suele aparecer junto con:

  • HTML, que define la estructura
  • CSS, que define la presentación visual
  • JavaScript, que define la lógica y el comportamiento

JavaScript permite:

  • guardar datos en variables
  • tomar decisiones
  • repetir acciones
  • agrupar instrucciones en funciones
  • trabajar con colecciones de datos
  • dividir código en módulos

Aunque muchas veces se lo asocia con el navegador, JavaScript también puede ejecutarse fuera de él, por ejemplo en servidores con Node.js.

¿Por qué es central en desarrollo web?

Porque permite programar la lógica de una aplicación:

  • validar datos
  • transformar información
  • calcular resultados
  • organizar código reutilizable
  • conectar distintas partes del sistema

En este apunte nos enfocaremos en los fundamentos del lenguaje. No veremos DOM ni eventos, ya que esos temas se tratarán por separado.

Formas de usar JavaScript en HTML

Existen dos formas habituales de incluir JavaScript en una página:

  1. dentro del mismo archivo HTML con <script>
  2. en un archivo .js separado

JavaScript dentro de <script>

Se puede escribir código JavaScript directamente dentro del documento HTML:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Ejemplo</title>
  </head>
  <body>
    <script>
      console.log("Hola desde JavaScript");
      let nombre = "Ana";
      console.log("Bienvenida, " + nombre);
    </script>
  </body>
</html>

Este enfoque puede ser útil para ejemplos pequeños o pruebas rápidas.

Sin embargo, tiene desventajas:

  • mezcla estructura con lógica
  • dificulta reutilizar código
  • complica el mantenimiento cuando el proyecto crece

JavaScript en archivo separado

La forma más recomendada en proyectos reales es escribir el código en un archivo aparte y enlazarlo desde HTML.

Por ejemplo, si tenemos este archivo app.js:

console.log("JavaScript cargado desde archivo externo");
 
let curso = "Programación III";
console.log(curso);

podemos conectarlo así:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Ejemplo</title>
  </head>
  <body>
    <script src="app.js"></script>
  </body>
</html>

¿Por qué conviene usar archivos separados?

  • separan contenido y lógica
  • facilitan mantenimiento
  • permiten reutilizar código
  • mejoran organización del proyecto
  • resultan más cómodos para trabajar en equipo

Variables y tipos de datos

Una variable es un nombre que referencia un valor.

let nombre = "Lucía";
let edad = 20;
let activo = true;

En JavaScript los valores pueden cambiar durante la ejecución del programa, y el lenguaje determina el tipo de dato en tiempo de ejecución.

Tipos de datos básicos

string

Representa texto.

let mensaje = "Hola";
let ciudad = 'Rosario';

number

Representa números enteros y decimales.

let cantidad = 10;
let precio = 99.5;

boolean

Representa verdadero o falso.

let estaAprobado = true;
let tieneErrores = false;

undefined

Indica que una variable existe, pero todavía no tiene valor asignado.

let resultado;
console.log(resultado); // undefined

null

Representa ausencia intencional de valor.

let usuarioActual = null;

object

Agrupa datos relacionados mediante pares clave-valor.

let alumno = {
  nombre: "Marcos",
  edad: 21
};

Otros tipos

También existen otros tipos menos usados en una introducción inicial, como bigint y symbol.

Declaración de variables: var, let y const

JavaScript ofrece tres formas principales de declarar variables:

var materia = "Programación";
let comision = "A";
const anio = 2026;

var

Es la forma más antigua.

var nombre = "Juan";
var nombre = "Pedro";

Características importantes:

  • tiene alcance de función, no de bloque
  • permite redeclarar la misma variable
  • puede reasignarse
  • su comportamiento puede generar errores difíciles de detectar

Ejemplo:

if (true) {
  var mensaje = "Hola";
}
 
console.log(mensaje); // "Hola"

Aunque mensaje fue declarada dentro del if, sigue existiendo fuera del bloque.

let

Se usa para variables cuyo valor puede cambiar.

let contador = 0;
contador = contador + 1;

Características:

  • tiene alcance de bloque
  • no permite redeclarar en el mismo bloque
  • sí permite reasignar
if (true) {
  let estado = "activo";
}
 
// console.log(estado); // Error: estado no existe fuera del bloque

const

Se usa cuando la referencia no debe cambiar.

const pi = 3.14159;

Características:

  • tiene alcance de bloque
  • no permite redeclarar
  • no permite reasignar
  • debe inicializarse al declararse
const curso = "Programación III";
// curso = "Otra materia"; // Error

Importante sobre const

Que una variable sea const no significa que el contenido del objeto o arreglo sea inmutable. Significa que la referencia no puede cambiar.

const alumno = {
  nombre: "Sofía"
};
 
alumno.nombre = "Martina"; // válido

Lo que no se puede hacer es esto:

const alumno = {
  nombre: "Sofía"
};
 
// alumno = { nombre: "Martina" }; // Error

Comparativa entre var, let y const

Característica var let const
Alcance función bloque bloque
Redeclaración en mismo bloque sí no no
Reasignación sí sí no
Inicialización obligatoria no no sí
Uso recomendado actual no sí sí

En código moderno, la práctica habitual es:

  • usar const por defecto
  • usar let cuando el valor debe cambiar
  • evitar var salvo que se esté leyendo o manteniendo código antiguo

Condicionales

Los condicionales permiten ejecutar distintos bloques de código según una condición.

if

let nota = 8;
 
if (nota >= 6) {
  console.log("Aprobado");
}

if / else

let edad = 17;
 
if (edad >= 18) {
  console.log("Mayor de edad");
} else {
  console.log("Menor de edad");
}

if / else if / else

let nota = 4;
 
if (nota >= 9) {
  console.log("Excelente");
} else if (nota >= 6) {
  console.log("Aprobado");
} else {
  console.log("Desaprobado");
}

switch

switch es útil cuando se quiere comparar una misma expresión contra varios casos posibles.

let dia = 3;
 
switch (dia) {
  case 1:
    console.log("Lunes");
    break;
  case 2:
    console.log("Martes");
    break;
  case 3:
    console.log("Miércoles");
    break;
  default:
    console.log("Otro día");
}

El break evita que la ejecución continúe en los siguientes casos.

Operador ternario

Es una forma breve de expresar una decisión simple.

let edad = 20;
let mensaje = edad >= 18 ? "Puede ingresar" : "No puede ingresar";
 
console.log(mensaje);

Suele ser conveniente para decisiones cortas. Si la lógica es más compleja, en general resulta más claro usar if.

Bucles

Los bucles permiten repetir instrucciones.

for

Se usa mucho cuando se conoce de antemano cuántas veces se quiere repetir algo.

for (let i = 0; i < 5; i++) {
  console.log("Vuelta número:", i);
}

Partes del for:

  1. inicialización
  2. condición
  3. actualización

while

Se repite mientras la condición sea verdadera.

let contador = 0;
 
while (contador < 3) {
  console.log(contador);
  contador++;
}

Si la condición nunca deja de cumplirse, se genera un bucle infinito.

do...while

Primero ejecuta el bloque y después evalúa la condición. Por eso, se ejecuta al menos una vez.

let numero = 5;
 
do {
  console.log("Valor:", numero);
  numero++;
} while (numero < 5);

En este caso el bloque se ejecuta una vez, aunque la condición ya sea falsa.

Funciones

Una función agrupa instrucciones reutilizables.

function saludar() {
  console.log("Hola");
}
 
saludar();

Funciones con parámetros

function saludarPersona(nombre) {
  console.log("Hola, " + nombre);
}
 
saludarPersona("Carla");

Funciones con retorno

function sumar(a, b) {
  return a + b;
}
 
let resultado = sumar(10, 5);
console.log(resultado);

Funciones flecha

Las funciones flecha son una sintaxis más corta para definir funciones.

const saludar = () => {
  console.log("Hola");
};

Con un parámetro:

const cuadrado = numero => {
  return numero * numero;
};

Si el cuerpo tiene una sola expresión, se puede usar retorno implícito:

const duplicar = numero => numero * 2;

Diferencias entre función tradicional y función flecha

1. Sintaxis

Función tradicional:

function sumar(a, b) {
  return a + b;
}

Función flecha:

const sumar = (a, b) => {
  return a + b;
};

2. this

Las funciones flecha no tienen su propio this. Toman el this del contexto exterior.

Las funciones tradicionales sí pueden tener su propio this, según cómo se las invoque.

Este tema se vuelve especialmente importante en objetos, clases y callbacks.

3. arguments

Las funciones tradicionales disponen del objeto especial arguments.

function ejemplo() {
  console.log(arguments);
}

Las funciones flecha no lo tienen.

4. Uso como constructor

Una función tradicional puede usarse con new en determinados contextos. Una función flecha no.

5. Hoisting

Las declaraciones de función suelen poder invocarse antes de aparecer en el código fuente:

saludar();
 
function saludar() {
  console.log("Hola");
}

En cambio, una función flecha asignada a const o let no puede usarse antes de la declaración:

// saludar(); // Error
 
const saludar = () => {
  console.log("Hola");
};

¿Cuál conviene usar?

Depende del caso:

  • función tradicional: cuando interesa una definición clásica, hoisting o un comportamiento propio de this
  • función flecha: cuando se busca sintaxis breve o capturar this del contexto exterior

Arreglos

Un arreglo almacena varios valores en una misma estructura.

let numeros = [10, 20, 30, 40];
let nombres = ["Ana", "Luis", "Mora"];
let mixto = [1, "texto", true];

Los elementos se acceden por índice, comenzando en 0.

let colores = ["rojo", "verde", "azul"];
 
console.log(colores[0]); // rojo
console.log(colores[2]); // azul

También se pueden modificar:

let frutas = ["manzana", "pera", "banana"];
frutas[1] = "naranja";

Recorrer un arreglo

let notas = [7, 8, 10];
 
for (let i = 0; i < notas.length; i++) {
  console.log(notas[i]);
}

Arreglos en línea

Muchas veces se crean arreglos directamente en el lugar donde se necesitan:

console.log(["HTML", "CSS", "JavaScript"]);

También pueden combinarse con objetos:

const curso = {
  nombre: "Programación III",
  temas: ["Git", "HTML", "CSS", "JavaScript"]
};

Objetos en línea

Un objeto permite agrupar datos mediante propiedades.

let alumno = {
  nombre: "Valentina",
  edad: 22,
  activo: true
};

Acceso a propiedades:

console.log(alumno.nombre);
console.log(alumno["edad"]);

También se pueden modificar o agregar propiedades:

alumno.edad = 23;
alumno.comision = "B";

Objetos en línea

Al igual que con los arreglos, un objeto puede definirse directamente en el lugar donde se lo necesita:

console.log({
  marca: "Ford",
  modelo: "Fiesta",
  anio: 2018
});

Arreglos y objetos combinados

Es muy común trabajar con estructuras combinadas:

const alumnos = [
  { nombre: "Ana", nota: 8 },
  { nombre: "Luis", nota: 6 },
  { nombre: "Mora", nota: 10 }
];
 
console.log(alumnos[0].nombre); // Ana
console.log(alumnos[2].nota);   // 10

Módulos

Cuando un programa crece, conviene dividirlo en varios archivos. En JavaScript, eso se logra con módulos.

Un módulo permite:

  • separar responsabilidades
  • reutilizar código
  • importar solo lo necesario
  • evitar mezclar todo en un único archivo

Exportar desde un módulo

Archivo matematica.js:

export const PI = 3.14159;
 
export function sumar(a, b) {
  return a + b;
}

Importar desde otro módulo

Archivo app.js:

import { PI, sumar } from "./matematica.js";
 
console.log(PI);
console.log(sumar(2, 3));

Conectar un módulo en HTML

Para que el navegador interprete el archivo como módulo, se usa:

<script type="module" src="app.js"></script>

Algunas características de los módulos

  • cada archivo tiene su propio alcance
  • para reutilizar algo, debe exportarse
  • para usar algo de otro archivo, debe importarse
  • type="module" habilita sintaxis import y export

Cierre

En esta introducción vimos los elementos básicos del lenguaje:

  • cómo incorporar JavaScript a una página
  • cómo declarar variables
  • qué tipos de datos básicos existen
  • cómo usar condicionales y bucles
  • cómo definir funciones
  • cómo trabajar con arreglos, objetos y módulos

Estos conceptos forman la base para avanzar luego hacia temas como manipulación del DOM, eventos, asincronía, consumo de APIs y desarrollo de aplicaciones más completas.
