Introducción a CSS

¿Qué es CSS?

CSS significa Cascading Style Sheets. Es el lenguaje que se utiliza para describir la presentación visual de un documento HTML.

Mientras que HTML define la estructura y el significado del contenido, CSS indica cómo debe verse ese contenido en pantalla.

Por ejemplo, con CSS se puede decidir:

  • qué color tendrá un texto
  • qué tipografía se utilizará
  • cuánto espacio habrá entre elementos
  • si un bloque tendrá borde, fondo o sombra
  • cómo se distribuirán los elementos en la página

¿Por qué conviene separar HTML y CSS?

Separar el contenido de la presentación trae varias ventajas importantes:

  • mejora el mantenimiento del proyecto
  • evita repetir estilos en muchos archivos
  • permite reutilizar una misma hoja de estilos en varias páginas
  • facilita cambiar el diseño sin tocar la estructura HTML
  • ayuda a mantener el código más ordenado y legible

En un proyecto pequeño puede parecer más rápido escribir estilos directamente en cada elemento, pero en cuanto el sitio crece esa estrategia se vuelve difícil de mantener.

Primer ejemplo

Supongamos el siguiente documento HTML:

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Ejemplo CSS</title>
  </head>
  <body>
    <h1>Programación III</h1>
    <p>Primer ejemplo con estilos.</p>
  </body>
</html>

Si agregamos la siguiente hoja de estilo:

body {
  font-family: Arial, sans-serif;
  background-color: #f4f4f4;
}
 
h1 {
  color: #1d4ed8;
}
 
p {
  color: #333333;
}

el contenido seguirá siendo el mismo, pero cambiará su apariencia.

La regla CSS

Una regla CSS está formada por un selector y un bloque de declaraciones.

En una regla como esta:

p {
  color: blue;
  font-size: 18px;
}
  • p es el selector
  • color y font-size son propiedades
  • blue y 18px son valores

Cada declaración termina en punto y coma, y el conjunto de declaraciones se escribe entre llaves.

Comentarios en CSS

Los comentarios permiten dejar aclaraciones dentro del archivo sin afectar el resultado visual.

/* Estilo principal del encabezado */
h1 {
  color: navy;
}

Formas de conectar CSS con HTML

Existen tres maneras habituales de aplicar CSS a una página.

  1. En línea
  2. Interno o integrado
  3. Externo

En línea

Se utiliza el atributo style dentro de una etiqueta HTML.

<p style="color: red">Un texto</p>

Este enfoque sirve para casos puntuales, pero tiene varias desventajas:

  • mezcla estructura con presentación
  • obliga a repetir estilos
  • dificulta el mantenimiento
  • no escala bien en proyectos medianos o grandes

Por eso, en general no se recomienda como estrategia principal.

  • ¿Cómo se podrían modificar todos los párrafos del archivo sin repetir style?

Interno o integrado

Se escribe CSS dentro de una etiqueta <style> ubicada en el <head>.

<head>
  <style>
    p {
      color: blue;
      font-size: 12px;
    }
  </style>
</head>

Este enfoque permite definir estilos para toda una página, pero sigue teniendo un límite: los estilos quedan atados a un único archivo HTML.

  • ¿Qué pasa si hay que aplicar esos mismos estilos en varias páginas?

Externo

La opción más usada es guardar las reglas CSS en un archivo separado y enlazarlo desde HTML con <link>.

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Ejemplo</title>
    <link rel="stylesheet" href="estilo.css" />
  </head>
  <body>
    ...
  </body>
</html>

Por ejemplo, el archivo estilo.css podría contener:

body {
  background-color: white;
  color: #222;
}

¿Por qué se prefieren los archivos CSS externos?

  • permiten reutilizar estilos en varias páginas
  • reducen repetición de código
  • facilitan el mantenimiento
  • ayudan a organizar mejor el proyecto
  • el navegador puede almacenar la hoja de estilos en caché

Sintaxis básica

La unidad fundamental de CSS es la regla:

selector {
  propiedad: valor;
  propiedad: valor;
}

Ejemplo:

h1 {
  color: green;
  text-align: center;
}

Selectores CSS

Los selectores indican a qué elementos HTML se aplicará una regla.

Algunos selectores básicos son:

  • selector de elemento
  • selector de clase
  • selector de id
  • selector universal
  • selector descendente
  • agrupación de selectores

Selector de elemento

Selecciona todos los elementos con un nombre determinado.

p {
  color: #444;
}
 
body {
  background-color: #fafafa;
}

Si el documento tiene diez párrafos, la regla p { ... } afectará a los diez.

Selector de id

Selecciona el elemento que tiene un atributo id específico.

<h1 id="encabezado">Este es un encabezado</h1>
<strong id="nombre">Este es un nombre</strong>
#encabezado {
  color: yellow;
}
 
#nombre {
  background-color: red;
}

Un id debería ser único dentro del documento. Se usa cuando un elemento tiene una identidad particular.

Selector de clase

Selecciona todos los elementos que comparten una clase.

<h1 class="titulo">Este es un título</h1>
<p class="direccion">Esta es una dirección</p>
.titulo {
  font-size: 36px;
}
 
.direccion {
  font-style: italic;
}

Las clases son muy importantes porque permiten reutilizar estilos.

Clase aplicada a un tipo de elemento

También se puede indicar que una clase afecte solo a cierto tipo de elemento.

<h1 class="titulo">Este es un título</h1>
<p class="titulo">Este texto también usa la clase titulo</p>
h1.titulo {
  font-size: 36px;
}

En este caso, la regla afecta únicamente a los <h1> que tengan la clase titulo, no a todos los elementos con esa clase.

Selector universal

El selector * selecciona todos los elementos.

* {
  font-size: 16px;
}

También puede combinarse con otros selectores:

section * {
  color: #333;
}

Esta regla afecta a todos los elementos que estén dentro de un <section>.

Selector descendente

Permite seleccionar elementos contenidos dentro de otros elementos.

<p>Esta es una <strong>palabra destacada</strong> dentro de un párrafo.</p>

p strong {
  background-color: green;
}

La regla no selecciona todos los <strong> del documento, sino solo los que están dentro de un <p>.

Agrupación de selectores

Si varios elementos deben compartir el mismo estilo, se pueden agrupar separándolos con comas.

p,
h1,
h2 {
  color: #1f2937;
}

Esto evita duplicar reglas.

Más de una clase en un elemento

Un mismo elemento puede tener varias clases.

<p class="texto importante">Atención: este mensaje es importante.</p>

.texto {
  font-family: Arial, sans-serif;
}
 
.importante {
  color: red;
  font-weight: bold;
}

Esto permite combinar estilos de forma flexible.

Pseudoclases básicas

Las pseudoclases permiten aplicar estilos según el estado de un elemento.

Por ejemplo:

a:hover {
  color: orange;
}
 
input:focus {
  border-color: blue;
}
  • :hover se activa cuando el puntero pasa sobre el elemento
  • :focus se activa cuando el elemento recibe el foco

Son muy usadas en enlaces, botones, formularios y menús.

Cascada, herencia y prioridad

La palabra cascading en CSS no es decorativa: describe cómo el navegador decide qué reglas aplicar cuando varias afectan al mismo elemento.

Para entender CSS con claridad conviene distinguir tres ideas:

  1. cascada
  2. herencia
  3. especificidad

Cascada

Puede ocurrir que varias reglas apunten al mismo elemento:

p {
  color: blue;
}
 
.destacado {
  color: red;
}
 
#principal {
  color: green;
}

Si un párrafo tiene clase destacado e id principal, varias reglas competirán por definir su color. El navegador debe resolver ese conflicto.

Herencia

Algunas propiedades se heredan automáticamente desde el elemento padre hacia sus hijos. Por ejemplo, el color y parte de las propiedades tipográficas.

body {
  color: #222;
  font-family: Arial, sans-serif;
}

Si no se define otra cosa, muchos elementos dentro del body heredarán esos valores.

No todas las propiedades se heredan. Por ejemplo, márgenes, bordes y anchos no suelen heredarse.

Prioridad general

De manera simplificada, el navegador asigna prioridad según el tipo de selector:

  1. estilos en línea
  2. selectores por id
  3. selectores por clase, atributo o pseudoclase
  4. selectores por elemento

Prioridad por especificidad

Cuanto más específica es una regla, mayor peso tiene.

.main .sale .clearance p {
  color: red;
}
 
.header .title p {
  color: green;
}
 
.footer p {
  color: blue;
}

En igualdad de condiciones, la regla más específica tiene prioridad.

Prioridad por posición

Si dos reglas tienen la misma especificidad, gana la que aparece más abajo en el archivo.

a {
  background-color: red;
}
 
a {
  background-color: yellow;
}
 
a {
  background-color: black;
}

El enlace terminará con fondo negro.

Ejemplo combinado

p {
  color: blue;
  font-family: Helvetica, sans-serif;
}
 
.rojo {
  color: red;
}
 
#especial {
  font-family: Arial, sans-serif;
}
<p>Párrafo normal</p>
<p class="rojo">Párrafo rojo</p>
<p class="rojo" id="especial">Párrafo rojo con otra fuente</p>

Valores CSS

Las propiedades CSS aceptan distintos tipos de valores. Dos grupos muy importantes al comenzar son:

  • colores
  • longitudes y porcentajes

Colores

El navegador admite varias formas de representar colores.

  • nombre del color, por ejemplo red
  • hexadecimal, por ejemplo #ff0000
  • RGB, por ejemplo rgb(255, 0, 0)
  • RGBA, por ejemplo rgba(255, 0, 0, 0.5)
  • HSL, por ejemplo hsl(0, 100%, 50%)

Ejemplo:

h1 {
  color: #0f172a;
}
 
body {
  background-color: rgb(240, 240, 240);
}

En la práctica, los formatos hexadecimal y RGB/RGBA son muy comunes.

Longitudes y porcentajes

CSS utiliza muchas unidades. Algunas de las más frecuentes son:

  • px: píxeles
  • %: porcentaje
  • em: relativo al tamaño de fuente del elemento
  • rem: relativo al tamaño de fuente del elemento raíz
  • pt: puntos, más común en contextos de impresión

Ejemplos:

h1 {
  font-size: 32px;
}
 
.contenedor {
  width: 80%;
}
 
p {
  line-height: 1.5;
}

Unidades absolutas y relativas

Conviene distinguir entre dos grandes grupos:

  • absolutas: como px, cm, mm, in
  • relativas: como %, em, rem

En interfaces web, las unidades relativas suelen aportar más flexibilidad, sobre todo cuando la página debe adaptarse a distintos tamaños de pantalla.

El valor cero

Cuando el valor es cero, no hace falta indicar la unidad:

margin: 0;
padding: 0;
border: 0;

Propiedades CSS frecuentes

Al comenzar, conviene reconocer algunas propiedades muy usadas.

Color y fondo

La propiedad color cambia el color del texto:

p {
  color: yellow;
}

La propiedad background-color cambia el color de fondo:

body {
  background-color: red;
}

También existen propiedades como:

  • background-image
  • background-repeat
  • background-position

pero para una primera aproximación suele alcanzar con background-color.

Texto y tipografía

Se puede modificar el aspecto del texto con distintas propiedades.

font-family

Indica la familia tipográfica que se intentará usar.

p {
  font-family: Arial, Helvetica, sans-serif;
}

El navegador intentará usar la primera fuente disponible. Si no la encuentra, probará con la siguiente.

Si el nombre de una fuente tiene espacios, debe escribirse entre comillas:

h1 {
  font-family: "Times New Roman", serif;
}

Otras propiedades tipográficas

  • font-size: tamaño del texto
  • font-weight: grosor de la fuente
  • font-style: estilo, por ejemplo cursiva
  • text-decoration: subrayado, tachado, etc.
  • text-transform: mayúsculas, minúsculas, capitalización

Ejemplo:

h2 {
  font-size: 24px;
  font-weight: bold;
  text-transform: uppercase;
}

Espaciado y alineación del texto

Otras propiedades útiles son:

  • letter-spacing: espacio entre letras
  • word-spacing: espacio entre palabras
  • line-height: altura de línea
  • text-indent: sangría de la primera línea
  • text-align: alineación horizontal del texto

Ejemplo:

p {
  line-height: 1.6;
  text-align: justify;
}

Buenas prácticas iniciales

Al comenzar a escribir CSS, conviene adoptar algunas costumbres sanas:

  • preferir hojas de estilo externas
  • usar nombres de clases claros y descriptivos
  • evitar estilos en línea salvo casos excepcionales
  • agrupar reglas relacionadas
  • no repetir estilos innecesariamente
  • mantener una indentación consistente

Por ejemplo, una clase como .boton-principal comunica mejor su intención que un nombre poco expresivo como .azul-grande.

Ejemplo completo

HTML

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <title>Ejemplo completo</title>
    <link rel="stylesheet" href="estilos.css" />
  </head>
  <body>
    <h1 class="titulo">Programación III</h1>
    <p class="descripcion">Introducción a HTML, CSS y JavaScript.</p>
    <a class="enlace" href="#">Ver más</a>
  </body>
</html>

CSS

body {
  font-family: Arial, sans-serif;
  background-color: #f3f4f6;
}
 
.titulo {
  color: #1d4ed8;
  text-align: center;
}
 
.descripcion {
  color: #374151;
  line-height: 1.5;
}
 
.enlace {
  color: #2563eb;
  text-decoration: none;
}

Este ejemplo muestra varias ideas al mismo tiempo:

  • uso de archivo CSS externo
  • aplicación de clases
  • manejo de color y tipografía
  • alineación de texto
  • personalización básica de un enlace

Resumen

CSS permite controlar la apariencia visual de un documento HTML.

En esta introducción aparecieron varios conceptos fundamentales:

  • qué rol cumple CSS en una página web
  • cómo se escribe una regla
  • cómo conectar CSS con HTML
  • qué tipos de selectores existen
  • cómo funcionan la cascada, la herencia y la especificidad
  • qué valores y unidades son frecuentes
  • qué propiedades básicas conviene conocer

A medida que se avance en desarrollo web, CSS incorporará nuevos temas como modelo de caja, display, Flexbox y Grid.
